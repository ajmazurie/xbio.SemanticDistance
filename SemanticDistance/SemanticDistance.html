SemanticDistance

When dealing with biological ontologies like Gene Ontology or EC an interesting information to get is the similarity between two terms. Imagine two genes annotated with a different GO term. Did they have a similar function or not ? Answering this question is not easy. A first crude approach to quantify the similarity of two terms in an ontology is to consider only their respective positions in this ontology (the distance between them being, for example, the number of nodes separating them); the SemanticDistance library is an implementation of a better approach proposed by Lord et al., itself based on previous works on semantic trees like WordNet.

The idea is here to use the notion of 'information content'; we can associate to each term of the ontology a value reflecting how informative this term is. The more the term is used, the less informative it is; the distance between two terms is then calculated by taking the information content of the parents terms they share. For example, if two terms only share the root of the ontology (the less informative of all terms), their distance is high. But if they share a rarely used parent, then their distance will be low (their similarity will be high). To obtain a complete (and better !) explanation of this algorithm you can either read the articles below or the annexe C of my PhD thesis (in French).

References:

  • Lord P, Stevens R, Brass A, Goble C: Investigating semantic similarity measures across the Gene Ontology: the relationship between sequence and annotation. Bioinformatics 2003, 19(10):1275-83.
  • Lin D: An Information-Theoretic Definition of Similarity. In Proceedings of the Fifteenth International Conference on Machine Learning, Morgan Kaufmann Publishers Inc. 1998:296-304.

Installation

SemanticDistance is a Python script, and require a working Python interpret to be installed. You just have to uncompress the provided archive in a given directory which is in your PYTHONPATH (see the Python documentation). SemanticDistance is then available from any Python script by simply importing it (import SemanticDistance).

Usage

Here is an example. Let's imagine a new ontology describing, for example, genes. The information you need are, for each term of the ontology, the list of all the parent terms, plus the number of times each term is used to annotate a gene. Here is the code:

from SemanticDistance import SemanticDistanceConstructor

# Declaration of the new ontology.
# Let's say we have 6 terms organized like this (A is the root):
#
#     ,--> B (5) -->.    ,--> D (5)
#    /               \  /
#  A (3)             C (2)
#    \               /  \
#     `--> E (2) -->'    `--> F (2)
#
# (the number being the usage of each term)

# First we create a structure that contains the usage of each term.
# Terms that are not used can be excluded.
usage = {
  'A': 3,
  'B': 5,
  'C': 2,
  'D': 5,
  'E': 2,
  'F': 2
 }

# Then we create a structure declaring the direct parents of each term.
# The ONLY node that must not be present in this structure is the
# root node (which have no parents by definition).
parents = {
  'B': ['A'],
  'C': ['B', 'E'],
  'D': ['C'],
  'E': ['A'],
  'F': ['C']
 }

# Finally, we use SemanticDistanceConstructor to calculate the
# information content of each node ...
sdc = SemanticDistanceConstructor(usage, parents)

sdc.find_ancestors()
sdc.calculate_ic()

# ... then to save it
sdc.save("example.data")

From now, calculating a distance between two terms of this ontology is pretty easy:

from SemanticDistance import SemanticDistanceLoader

# First, loading the information generated for this ontology
sd = SemanticDistanceLoader("example.data")

# You can then calculate the distance between two terms by doing
print sd.distance_between('B', 'B') # return 0.0 (the lowest distance)
print sd.distance_between('B', 'C') # return 0.419755281904
print sd.distance_between('D', 'C') # return 0.282289068435
print sd.distance_between('B', 'E') # return 1.0 (the highest distance)

# Let's imagine we have a gene annotated with 'B', 'C' and 'D'.
# You can ...

# ... get the ancestor of these three terms that have the
# best information content (i.e. the most informative one)
print sd.best_common_ancestor(['B', 'C', 'D']) # return 'B'
print sd.best_common_ancestor(['C', 'D', 'F']) # return 'C'

# ... obtain the dispersion of the annotation (average distance
# between the terms)
print sd.dispersion(['B', 'C', 'D']) # return 0.443238197585

# Now let's imagine you have two genes, one annotated with 'B',
# the other with 'C' and 'D'. To know the distance between them,
print sd.distance_between_sets(['B'], ['C', 'D']) # return 0.489060268741

(these examples are available in the example.py file)

Gene Ontology and EC

Two ontologies are frequently used in biology to annotate genes: Gene Ontology and EC. SemanticDistance is provided with two scripts, WhereToGO and WhatTheEC, that automatize the generation of the information content for them. An additional library, Helpers, is also provided to deal with specific question about these ontologies.

Gene Ontology

To generate the information content for the terms in Gene Ontology, you need to retrieve and uncompress a file from the Gene Ontology database named go_YYYYMM-assocdb-tables.tar.gz (that you can find here), where YYYY and MM are the year and month release, respectively. The last version tested to work with WhereToGO is the one of August 2006.

Once this file decompressed in a directory (for example, 'go_files'), you can launch WhereToGo by typing on the shell ./WhereToGO.py --go-directory go_files. The optional --without-iea option makes GO annotations associated to a gene or protein with an IEA (inferred by electronic annotation) to be ignored.

from SemanticDistance import GoHelper

goH = GoHelper("go.terms")

# ... obtaining a list of all the GO terms of the ontology
# (three aspects mixed)
print goH.get_terms()

terms = [
  'GO:0000166', 'GO:0004674', 'GO:0004705', 'GO:0004707',
  'GO:0005515', 'GO:0005524', 'GO:0006468', 'GO:0006928',
  'GO:0006950', 'GO:0007165', 'GO:0007254', 'GO:0016740'
 ]

# ... classifying a given list of GO terms by aspect
print goH.classify(terms)
# return
#  {'molecular_function': ['GO:0000166', ...],
#   'cellular_component': [],
#   'biological_process': ['GO:0006468', ...]}

# ... or obtaining the name of a given term
print goH.get_name("GO:0004672") # return 'protein kinase activity'

Note: not all the GO terms returned by the get_terms() function are guaranteed to have an information content. In order to use this function in combination with SemanticDistance (for example, to calculate a distance between any pair of term in the GO ontology), you would like to do this:

all_terms = goH.get_terms()

# then we filter to keep the ones with an I.C.
terms_with_ic = sd.filter_terms(all_terms)

# two terms from two different aspects with result
# by a NO_SHARED_PARENT error)

EC

The use of EC will require you to download two files, enzyme.dat and enzclass.txt from the ExPASy database (you can obtain them here). Place these files in a given directory, for example 'ec_files'.

After using WhatTheEC (./WhatTheEC.py --ec-directory ec_files) you obtain two files, ec.data and ec.terms)

The EcHelper library can be used to retrieve the name of an EC term:

from SemanticDistance import EcHelper

ecH = EcHelper("ec.terms")

print ecH.get_name("2.7.1.-")
# return 'Transferases transferring phosphorous-containing groups
# phosphotransferases with an alcohol group as acceptor'

print ecH.get_name("2.7.-.-")
# return 'Transferases transferring phosphorous-containing groups'

Limitations

The principal limitations of SemanticDistance are the arbitrary choices made when dealing with the ontologies; according to the papers of Lord et al., all semantic links between terms are considered as 'is-a' links, and the distance between these terms is calculated by using the Lin metric. Of course you can modify the source to adapt any other metric (like Resnik or Jiang ones). This should be just fine for most of the applications of SemanticDistance, especially when used with Gene Ontology.
